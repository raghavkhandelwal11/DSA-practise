# What is time complexity

    wrong definition: It is time taken to run the code.

    correct definition: Rate of increase in time w.r.t to input in size.

    Time complexity = Big oh Notation => O()

    for i in range(1, 6):
        print("Raghav");


    initialize i
    check condiition
    print i
    check condition again
    print i

    like this it goes for n iterations that why its time complexity is O(n)


    Note: 
        1. Always Calculate time complexity in worst case.
        2. Avoid the constant values
        3. Avoid lower bound

    There are there cases for time complexity.
        1. Best case or lower bound  Ω()
        2. Average Case θ()
        3. Worst Case or Upper bound O()
        
    example

    for i in range(1, n+1):
        for j in range(1, i+1):
        pass

    in the above code let us see for how many times it runs.

    for i = 1, j will run for 1 time
    for i = 2, j will run for 2 times
    for i = 3, j will run for 3 times
    and so on 

    we can say for i = n, j will run for n times.

    in total the code inside will run for 1 + 2 + 3 + 4 + .... N times

    which means total n(n+1)/2 times
    simplifying this we get n2 as biggest term so the time complexity would be O(n^2)

# Time complexity for common python list methods

    method->time complexity O()
    copy->O(n)
    append->O(1)
    pop last->O(1)
    pop intermidiate->O(n)
    insert->O(n)
    get item->O(1)
    set item->O(1)
    delete item->O(n)
    iteration->O(n)
    get slice->O(k) where k is the length of slice
    del slice->O(n)
    set slice->O(k+n)
    sort->O(n log(n))
    multiply->O(nk)
    min(s), max(s) -> O(n)
    len(list)->O(1)
    x in s-> O(n)

    note:
        In a set x in s operation has average case O(1) and worst case O(n)

        myset = {5, 3, 2, 1, 10, 11, 15}

        15 in myset
